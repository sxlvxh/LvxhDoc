新增活动
这里是接口描述
请求路径：ppprocess/addActivity.action
请求参数描述
属性 类型 备注 说明
caseId int 必须输入 案件ID
lastActivityId int 必须输入 案件的最近一次活动的ID（对应pp_case.last_Activity_Id）
activityType int 必须输入 活动类型（对应pp_process.CASE_PROCESS_STEP）
1. 接访登记 3. 接访 4. 立案侦查 6. 询问 7. 起诉 9. 出庭 10. 结案
activityName String 必须输入 活动名称
localRoomId int 必须输入 本地媒体室ID
remoteRoomId int 可选 远程媒体室ID
clerkDepId int 必须输入 书记员所属部门的部门ID
clerkId int 必须输入 书记员的用户ID
clerkName String 必须输入 书记员姓名
undertakerEntCode String 必须输入 承办人所属单位的单位编码
undertakerDepId int 必须输入 承办人所属部门的部门编码
undertakerUserId int 必须输入 承办人的用户ID
undertakerName String 必须输入 承办人的姓名
请求JSON： 
{
  "caseId": 55,
  "activityType": 3,
  "activityName": "接访！",
  "localRoomId": 10100074,
  "lastActivityId": 30,
  "clerkId": 13018625,
  "clerkDepId": 320101,
  "clerkName": "苏Atest001",
  "undertakerEntCode": "320101",
  "undertakerDepId": 320101,
  "undertakerUserId": 1,
  "undertakerName": "yjp"
}
返回参数描述：
属性 类型 备注 说明
code int 返回码 0：成功 1：失败
desc String 返回描述
返回JSON： 
{
    "code": 0,
    "result": 31
}

一、            操作系统安装

1.       安装SUSE12操作系统，详见《操作系统安装》。

 

二、            初始化系统环境

1.       将deploy_basic.run上传到服务器的home目录下，并给文件赋予执行权限并安装，该包主要初始化系统环境和一键安装mysql服务

2.       #chmod 755 deploy_basic.run-----------赋予执行权限

3.       #./deploy_basic.run-----------------------执行安装，初始化系统环境

4.       #mysql –u root –p -------------------------输入密码Huaiye@2013**，检查mysql是否安装成功

三、            安装SIE

1.       将deploy_sie_sles12.run上传到服务器的home目录下，并给文件赋予执行权限并安装，该包主要是安装sdedb数据库和一键安装SIE服务

2.       #chmod 755 deploy_sie_sles12.run-------------赋予执行权限

3.       #./deploy_sie_sles12.run--------------------------执行安装，安装sdedb数据库和SIE服务

4.       #ps -ef |grep hy_media_server-----------------------------------------检查SIE服务是否成功

5.       进入/home/hy_media_server/bin目录下，执行命令停止SIE服务，

#./stop.sh--------------------------------------停止SIE服务(检查确保停止了服务)

#. Profile.sh------------------------------------初始化环境

# ./formatdisk.sh-----------------------------首次安装需要格式化裸盘，用来存储录制的视频

# ./formatool -s 1048576000 -b 16384 -r 0 -p /FILEdata/HyPicture.dat--------新建文件存储语音和上传的图片备注：要将格的块加到数据库中，在文档后面数据库表中说明；

      需要注意检查有此文件若无则可以自行建立，并且赋予权限

PS:流媒体存储录像失败：1、服务器时间不正确

2、t_sdp_param_config表中NOTIFY_RECORD_URL字段不正确

 

四、            安装ECS服务  

1.       使用root用户将setup.zip 和ecs-war.war 上传到home目录

2.       #unzip setup.zip---------------------------------------解压setup.zip包

3.       #cd setup------------------------------------------------进入目录

4.       #chmod 755 *-------------------------------------------赋予执行权限

5.       # ./MvsInstall_Prepare.sh-----------------------------创建相关用户

  

6.       #./DBInstall_ForMySql5.6.sh--------------------------创建ecsdb库

                                                                                                                                           

7.       # ./MvsInstall_ForAll.sh--------------------------------------解压ecs-war包

  

五、   安装meeting_server会议服务

1.       将deploy_hms_x264_sles12.run上传到服务器home目录下，并赋予执行权限

2.       #chmod 755 deploy_hms_x264_sles12.run---------------------赋予执行权限

3.       ./ deploy_hms_x264_sles12.run-----------------------------------执行安装

4、确认生成/home/hy_meeting_server文件夹

八、无固定IP服务的下级域（移动指挥车）模式(适用于基站)

老版本的平台配置：(SIE的版本不一样，配置也不一样，当前我们的版本是UMP_V200R002B11SP04_release，所以版本的配置如下)

SIE下级域配置自身的域，类型为2，主动注册域

下级域t_turn_config表中配置上级域的地址（如果内网相通，直接是内网地址，如果不通，填写上级域的外网地址）

如果上级域有内外网，下级域访问上级域需要将t_sdp_param_config里面的INTRANET_CONNECT配置成0

注意：一旦配置了INTRANET_CONNECT为0，那么就会从直系上级域获取下发的DOMAIN_NAT_IP,使用这个IP向上级域注册。因此，上级域配置的DOMAIN_NAT_IP须确保是一个可正常连接使用的外网IP。

t_plugin表中的插件：

 

上级域也是只配置自身（不要配置下级域）：域类型就是普通域

ECS上下级域的业务端配置：（业务端的配置和以前一样，只是在配置下级域的时候有一个字段选择“是否TCP服务”，选择“是”，上级域的conf_ent和下级域的conf_ent表格中的这个字段都是1）

HMS下级域的t_turn_config表中配置上级域的地址（如果内网相通，直接是内网地址，如果不通，填写上级域的外网地址）

 

PS：netstat -an | grep ':8282' 用这个可以查询 下级域是否连上了

ESTABLISHED 表示连接了

 

 

（新版）调测多级多域的配置：（老版本和新版本之间的区别在：t_domain_info表格中域类型多了一个“注册上级域”。新版本是UMP_SIE_V200R002B25）

 

自身是普通域：t_domain_info表配置可直连的域和自身域，域类型是普通（取值0）（这条数据在run包安装时已经预置，可按需修改域code）。不需手工配置任何注册4G域：如果有注册域向它注册，通过VMS域配置页面的下方可发现当前注册域。将必要的注册域添加到域信息表中，注册域类型为“主动注册域”（取值2）。

 

自身是注册4G域：修改t_domain_info表配置自身域的域类型为“主动注册域”（取值2），且自己的parent_domaincode填写直系上级域。 同时将直系上级域的外网IP和端口（3478）配置到t_turn_config表。这个IP端口也就是直系上级域的配置的外网地址（即上级域domain表配置的DOMAIN_NAT_IP）。同时需要通过VMS域配置页面的下方，将所有的注册上级域添加到域信息表中，域类型为“注册上级域”（取值4）。

 

注意：所有的需要进行互通消息的域，都需要添加配置到域信息表t_domain_info中，包括注册域，否则域之间无法互通。

PS：不管平台的的配置如何，业务端的配置和以前一样，上级域的conf_ent表格中的下级域和下级域的conf_ent表格中下级域本身的是否TCP服务字段都是1。

 

九、文件服务器部署与配置

备注：下级域的文件服务的地址要配置成上级域的文件服务地址

【必做】部署

将安装包ECSFileServer-0.0.1-SNAPSHOT.war重命名为ECSFileServer.war，再将ECSFileServer.war复制到Tomcat安装目录下的webapps子目录下。然后重新启动Tomcat。

【必做】验证文件服务器是否部署成功。

在浏览器中输入如下URL：

http://ip:port/ECSFileServer/index.jsp

 

其中，ip和port分别为文件服务器所做web服务器（Tomcat）的IP地址和端口号。

 

页面上显示“Hello World!”表示部署成功。

 

 

【可选】根据需要修改文件服务器配置参数

配置文件WEB-INF/classes/application.properties

# max single file size allowed to be uploaded. Unit:Bytes

upload.image.max_single_size = 10485760

# max total file size allowed to be uploaded. Unit:Bytes

upload.image.max_total_size = 31457280

#File upload whitelist

upload.image.white_list=jpg|jpeg|png|gif|bmp

 

该文件中的各个配置项的说明如下：

 

配置项

默认值

用途

备注

upload.{File_TYPE}.max_single_size

10485760

用于限制允许上传的单个文件的最大大小。单位：字节

{File_TYPE}表示文件upload类型。比如，“image”代表图片文件，”soft”软件包。

upload.{File_TYPE}.max_total_size

31457280

批量上传文件时，用于限制一次上传所允许的所有文件的最大总大小。单位：字节

upload.{File_TYPE}.white_list

“”

文件上传白名单。用于指定特定类型上传（比如图片文件上传）所允许的文件扩展名。只有扩展名在该列表中文件才允许上传。””表示不开启指定类型上传的白名单功能。
